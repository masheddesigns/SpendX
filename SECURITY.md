# Security Policy

## Supported Versions

We support security updates for the following versions of SpendX:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within SpendX, please do not disclose it publicly. We want to keep SpendX secure for all users.

To report a vulnerability:
1. Open a private issue on GitHub (if available) or send a detailed email to the project maintainers.
2. Please provide:
   - A description of the vulnerability.
   - The impact of the vulnerability.
   - Steps to reproduce (including a proof-of-concept, if possible).

We will acknowledge your report within 48 hours and work with you to patch and release a fix as quickly as possible.
